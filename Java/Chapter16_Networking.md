Networking
----------

-	네트워킹이란 두 대 이상의 컴퓨터를 케이블로 연결하여 네트워크를 구성하는 것이다.<br><br>
-	java.net 패키지를 사용하면 네트워크 어플리케이션의 데이터 통신 부분을 쉽게 작성할 수 있다.<br><br>

Client & Server
---------------

-	서버는 서비스를 제공하는 컴퓨터이며, 클라이언트는 서비스를 사용하는 컴퓨터이다.<br><br>
-	서비스는 서버가 클라이언트로부터 요청받은 작업을 처리하여 그 결과를 제공하는 것을 뜻하며, 서비스 종류에 따라 파일 서버와 메일 서버 및 어플리케이션 서버 등이 있다.<br><br>
-	하나의 서버가 여러 가지 서비스를 제공하기도 하며, 하나의 서비스를 여러 대의 서버로 제공하기도 한다.<br><br>
-	서버가 서비스를 제공하기 위해서는 서버 프로그램이 있어야 하며, 클라이언트가 서비스를 제공받기 위해서는 서버 프로그램과 연결할 수 있는 클라이언트 프로그램이 있어야 한다.<br><br>
-	네트워크를 구성할 때 전용 서버를 두는 것을 서버 기반 모델이라고 하며, 각 클라이언트가 서버 역할을 동시에 수행하는 것을 P2P 모델이라고 한다.<br><br>

IP Address
----------

-	IP 주소는 컴퓨터(host)를 구별하는데 사용되는 고유한 값으로, 인터넷에 연결된 모든 컴퓨터는 IP 주소를 갖는다.<br><br>
-	IP 주소는 4 byte의 정수로 구성되어 있으며, 4개의 정수가 마침표를 구분자로 표현된다.<br><br>
-	4개의 정수는 부호없는 1 byte 값인 0~255 사이의 정수이다.<br><br>
-	IP 주소는 네트워크 주소와 호스트 주소로 나뉜다.<br><br>
-	IP 주소와 서브넷 마스크를 비트 연산자 &로 연산하면 네트워크 주소만을 추출할 수 있다.<br><br>
-	IP 주소에서 네트워크 주소가 차지하는 자리수가 많을 수록, 호스트 주소의 범위가 줄어들기 때문에 네트워크의 규모가 작아진다.<br><br>
-	호스트 주소가 0인 것은 네트워크 자신을 나타내며, 255는 브로드캐스트 주소로 사용된다.<br><br>
-	서로 다른 두 호스트의 IP 주소의 네트워크 주소가 같다는 것은, 두 호스트가 같은 네트워크에 포함되어 있다는 것을 의미한다.<br><br>

URL(Uniform Resource Locator)
-----------------------------

-	URL은 인터넷에 존재하는 여러 서버들이 제공하는 자원에 접근할 수 있는 주소이다.<br><br>
-	"프로토콜://호스트명:포트번호/경로명/파일명?쿼리스트링#참조" 형태로 이루어져 있다.<br><br>
-	포트 번호와 쿼리 및 참조는 생략이 가능하다.<br><br>
-	URL 클래스는 URL을 다루는 여러 메서드를 제공한다.<br><br>

URL Connection
--------------

-	URL간의 통신 연결을 나타내는 클래스의 최상위 클래스로 추상 클래스이다.<br><br>
-	이를 상속받아 구현한 클래스로는 HttpURLConnection 클래스와 JarURLConnection 클래스가 있다.<br><br>
-	`openConnection()` 메서드는 URL의 프로토콜이 http 프로토콜일 경우, HttpURLConnection을 반환한다.<br><br>
-	URLConnection을 사용하여 연결하고자 하는 자원에 접근하여 읽고 쓰기를 할 수 있다.<br><br>
-	`openStream()` 메서드로 URL의 InputStream을 얻은 이후는 파일 데이터를 읽는 것과 다름없다.<br><br>

소켓 프로그래밍
---------------

-	소켓 프로그래밍은 소켓을 이용한 통신 프로그래밍으로서, 소켓은 프로세스간의 통신에 사용되는 양쪽 끝단을 의미한다.<br><br>
-	프로세스간의 통신을 위해서는 소켓이 필요하다.<br><br>
-	소켓 통신에 사용되는 프로토콜에 따라서 다른 종류의 소켓을 구현하여 제공한다.<br><br>

TCP & UDP
---------

![image](https://user-images.githubusercontent.com/56240505/72355380-b13de880-372a-11ea-8b22-b78251a42a35.png)<br><br>

-	TCP/IP 프로토콜은 이기종 시스템간의 통신을 위한 표준 프로토콜로, 프로토콜의 집합이다.<br><br>
-	TCP와 UDP 모두 TCP/IP 프로토콜에 속하며, OSI 7계층의 전송 계층에 해당하는 프로토콜이다.<br><br>
-	두 프로토콜은 전송 방식이 다르며, 장단점이 있다.<br><br>

TCP 소켓 프로그래밍
-------------------

-	TCP 소켓 프로그래밍은 클라이언트와 서버간의 일대일 통신이다.<br><br>
-	1. 서버 프로그램에서는 서버 소켓을 사용해 서버 컴퓨터의 특정 포트에서 클라이언트의 연결 요청을 처리할 준비를 한다.<br><br>
-	2. 클라이언트 프로그램은 접속할 서버의 IP 주소와 포트 정보를 가지고 소켓을 생성해서 서버에 연결을 요청한다.<br><br>
-	3. 서버 소켓은 클라이언트의 연결 요청을 받으면 서버에 새로운 소켓을 생성해서 클라이언트의 소켓과 연결되도록 한다.<br><br>
-	4. 클라이언트의 소켓과 새로 생성된 서버의 소켓은 서버 소켓과 관계 없이 일대일 통신을 한다.<br><br>
-	서버 소켓은 포트와 결합되어 포트를 통해 원격 사용자의 연결 요청을 기다리다가, 연결 요청이 올 때마다 새로운 소켓을 생성하여 상대편 소켓과 통신할 수 있도록 연결한다.<br><br>
-	실제 데이터 통신은 서버 소켓과 관계없이 소켓과 소켓 간에 이루어진다.<br><br>
-	서버 소켓은 전화 교환기에, 소켓은 전화기에 비유할 수 있다.<br><br>
-	서버 소켓은 포트를 독점한다.<br><br>
-	포트는 호스트가 외부와 통신을 하기 위한 통로로, 하나의 호스트가 65536개의 포트를 가지고 있으며 포트는 번호로 구별된다.<br><br>
-	보통 1023번 이하의 포트는 같은 기존의 다른 통신 프로그램들에 의해서 사용되는 경우가 많아, 그 이상의 번호 중 사용하지 않는 포트를 사용해야 한다.<br><br>
-	두 서버 소켓이 서로 다른 프로토콜을 사용하는 경우, 같은 포트를 사용할 수 있으나 지양한다.<br><br>
-	소켓들이 데이터를 주고 받는 연결 통로는 입출력 스트림이다.<br><br>
-	한 소켓의 입력 스트림과 출력 스트림은 각각 상대편 소켓의 출력 스트림과 입력 스트림으로 연결된다.<br><br>
-	Socket 클래스는 프로세스간의 통신을 담당하며, 입출력 스트림을 가지고 있다.<br><br>
-	ServerSocket 클래스는 포트와 연결되어 외부의 연결 요청을 기다리다 연결 요청이 들어오면, Socket을 생성해서 소켓간의 통신이 이루어지도록 한다.<br><br>
-	한 포트에 하나의 ServerSocket만 연결할 수 있다.<br><br>

TCP Practice
------------

```java
ServerSocket serverSocket = new ServerSocket(7777);
Socket socket = serverSocket.accept();
Socket clientSocket = new Socket("192.168.10.10", 7777);
//연결 요청이 accept되면 socket과 clientSocket끼리 스트림을 통해 통신 가능
```

-	서버 소켓이 특정 포트를 사용하고 있어도, 서버 소켓이 아닌 소켓은 해당 포트를 사용할 수 있다.<br><br>
-	클라이언트 프로그램의 소켓이 사용하는 포트는 사용 가능한 임의의 포트가 선택된다.<br><br>
-	쓰레드를 사용하면 멀티 채팅 서버 프로그램을 생성할 수 있다.<br><br>

UDP 소켓 프로그래밍
-------------------

-	UDP 소켓 프로그래밍은 DatagramSocket 클래스와 DatagramPacket 클래스를 사용한다.<br><br>
-	UDP는 연결 지향 프로토콜이 아니기 때문에 ServerSocket이 필요없다.<br><br>
-	UDP 통신에서 사용하는 소켓은 DatagramSocket이며, 데이터를 DatagramPacket에 담아 전송한다.<br><br>
-	DatagramPacket은 헤더와 데이터로 구성되어 있으며, 헤더에는 DatagramPacket을 수신할 호스트의 주소 및 포트 등의 정보가 저장되어 있다.<br><br>
